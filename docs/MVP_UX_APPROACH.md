# Specifai MVP - UX 접근 방향성

**작성일:** 2025년 11월 12일  
**버전:** 1.0  
**목적:** 핵심 기능 검증을 위한 MVP UX 설계 방향 정립

---

## 0. 기본 사용자 프로세스

### 0.1 전체 프로세스 흐름

Specifai의 기본 사용자 프로세스는 다음과 같은 단계로 진행됩니다:

```
서비스 접속
    ↓
초안 작성 (프로젝트/기능 설명)
    ↓
AI 검토 및 분석
    ↓
질의응답 시작
    ↓
대주제 A 선택
    ↓
(필요 시) 소주제 선택
    ↓
대주제 B 선택
    ↓
... 반복 ...
    ↓
질의응답 종료
    ↓
AI 최종 검토
    ↓
요구사항 명세서 생성
```

### 0.2 단계별 상세 설명

#### Step 1: 서비스 접속
- 사용자가 Specifai 웹사이트에 접속
- (MVP) 별도 회원가입 없이 바로 시작 가능

#### Step 2: 초안 작성
- 사용자가 구현하려는 프로젝트 또는 기능에 대해 자유롭게 작성
- 예시: "온라인 쇼핑몰을 만들고 싶어요. 상품을 등록하고 판매할 수 있으면 좋겠어요."
- 짧은 설명(1-2줄)부터 긴 설명(여러 문단)까지 모두 가능

#### Step 3: AI 검토
AI가 초안을 분석하여 다음 항목들을 파악하고, 사용자에게 확인합니다:

**3-1. 프로젝트 유형 (Project Type)**
- AI가 자동으로 분류하는 카테고리:
  - `이커머스 (쇼핑몰)`: 상품 판매, 주문, 결제가 핵심
  - `커뮤니티`: 게시글, 댓글, 사용자 상호작용이 핵심
  - `SaaS (서비스형 소프트웨어)`: 특정 비즈니스 기능 제공
  - `랜딩 페이지`: 정보 제공 및 전환이 목적
  - `관리자 대시보드`: 데이터 관리 및 모니터링
  - `예약/예매 시스템`: 일정 관리 및 예약이 핵심
  - `교육 플랫폼`: 강의, 학습 콘텐츠 제공
  - `콘텐츠 관리 시스템 (CMS)`: 다양한 콘텐츠 관리
  - `기타`: 위 카테고리에 해당하지 않는 프로젝트

**3-2. 핵심 기능 (Core Features)**
- 초안에서 명시적으로 언급된 기능 식별:
  - 예: "회원 관리", "상품 등록", "결제", "검색", "댓글", "알림" 등
- 암묵적으로 필요한 기능 추론:
  - 예: 쇼핑몰 → 장바구니, 주문 관리 필요 가능성 높음

**3-3. 예상 복잡도 (Estimated Complexity)**
- 파악된 정보를 바탕으로 프로젝트 규모 추정:
  - `단순 (10-15개 질문, 10분 소요)`:
    - 랜딩 페이지
    - 간단한 CRUD 게시판
    - 정보 제공 웹사이트
  - `중간 (15-25개 질문, 15-20분 소요)`:
    - 소규모 쇼핑몰
    - 커뮤니티 사이트
    - 기본 예약 시스템
  - `복잡 (25-40개 질문, 20-30분 소요)`:
    - 다중 역할 플랫폼
    - 결제 시스템 포함
    - 복잡한 권한 관리
  - `매우 복잡 (40-50개 질문, 30-45분 소요)`:
    - 엔터프라이즈 솔루션
    - 다중 사용자 그룹
    - 고급 비즈니스 로직

**3-4. 프로젝트 목적 (Project Purpose)**
- 사용자가 이 프로젝트를 만드는 근본적인 이유:
  - `신규 수익 창출`: 새로운 비즈니스 모델
  - `기존 프로세스 개선`: 업무 효율화, 자동화
  - `정보 제공`: 콘텐츠 전달, 브랜딩
  - `사용자 확보`: 커뮤니티 구축, 플랫폼 성장
  - `내부 도구`: 팀/조직 내부용

**3-5. 대상 사용자 (Target User)**
- 서비스를 이용할 주요 사용자 그룹:
  - `일반 소비자 (B2C)`: 개인 사용자
  - `기업 고객 (B2B)`: 비즈니스 사용자
  - `콘텐츠 제공자`: 판매자, 작가, 크리에이터
  - `관리자/운영자`: 서비스 운영 담당자
  - `내부 직원`: 사내 직원용

**AI 분석 결과 확인 화면 예시:**
```
📋 AI가 파악한 주요 내용

프로젝트 유형: 이커머스 (쇼핑몰)
예상 복잡도: 중간 (약 20-25개 질문, 15-20분 소요)

파악된 핵심 기능:
  ✓ 상품 등록 및 관리
  ✓ 회원 관리 (추정)
  ✓ 주문 및 결제 (추정)
  
프로젝트 목적: 신규 수익 창출 (추정)
대상 사용자: 일반 소비자 (B2C) (추정)

이 내용이 맞나요?
[✓ 네, 맞습니다] [수정이 필요해요]
```

이렇게 파악된 정보는 이후 질의응답 전략을 수립하는 기반 데이터로 사용됩니다.

#### Step 4: 질의응답 시작
- AI가 체계적인 질문을 시작
- 대주제(카테고리) 기반으로 진행

#### Step 5-7: 대주제 및 소주제 선택
- **대주제 A 선택**: 예) "사용자 인증" 대주제에서 항목 선택
  - 회원가입 방식: `이메일 가입` 선택
  - 소셜 로그인: `카카오`, `네이버` 선택 (다중 선택)
  
- **소주제 선택 (조건부)**:
  - 이전 답변에 따라 추가 세부 질문 표시
  - 예) 소셜 로그인 선택 시 → "어떤 소셜 로그인을 원하시나요?"

- **대주제 B 선택**: 예) "핵심 데이터 관리" 대주제로 이동
  - 핵심 콘텐츠: `상품` 선택
  - 생성 권한: `관리자만` 선택

#### Step 8: 반복
- 모든 필수 대주제를 순회하며 질의응답 진행
- 프로젝트 복잡도에 따라 10-50개 질문

#### Step 9: 질의응답 종료
- 모든 필수 질문 완료
- 사용자가 최종 확인 및 수정 가능

#### Step 10: AI 최종 검토
- AI가 수집된 답변을 종합 분석:
  - 누락된 항목 확인
  - 모순된 요구사항 검증
  - 애매한 표현 명확화
  - 필요 시 추가 질문

#### Step 11: 요구사항 명세서 생성
- **표준 SRS 문서** 생성 (계약용)
- **AI-Native Instruction Prompt** 생성 (AI 코딩 도구용)
- PDF 다운로드 및 공유 가능

### 0.3 프로세스 예시 (쇼핑몰 프로젝트)

```
[Step 1-2] 사용자 입력:
"온라인 쇼핑몰을 만들고 싶어요. 관리자가 상품을 등록하고, 
고객이 구매할 수 있으면 됩니다."

[Step 3] AI 분석:
✓ 프로젝트 유형: 이커머스
✓ 복잡도: 중간
✓ 예상 질문 수: 20-25개

[Step 4-8] 질의응답:

  A: ☑ 이메일 가입
  
  Q: 소셜 로그인이 필요한가요?

→ 사용자: ☑ 예
```

### 0.4 프로세스 설계 핵심 원칙

1. **점진적 구체화**: 초안 → 대주제 → 소주제 순으로 세부화
2. **조건부 질문**: 이전 답변에 따라 필요한 질문만 표시
3. **객관식 중심**: 90% 이상 선택형, 주관식 최소화
4. **실시간 미리보기**: 질의응답 중 명세서 초안 실시간 업데이트
5. **양방향 수정**: 언제든 이전 단계로 돌아가 수정 가능

## 1. 핵심 UX 원칙

### 1.1 기본 원칙

```
사용자 입력 → 객관식 선택 중심 → 빠른 결과 도출
```

**목표:**
- ✅ 최초 요구사항 입력 후 **객관식 선택만으로** 명세서 완성
- ✅ 질문 개수: **20개 이내** (기본), 최대 50개 (복잡한 프로젝트)
- ✅ 완료 시간: **10-30분** 이내
- ✅ 비개발자도 쉽게 답변 가능

### 1.2 동적 질문 흐름

프로젝트의 **규모, 주제, 구체화 수준**에 따라 질문 개수와 단계가 자동 조정됩니다.

| 프로젝트 유형 | 질문 개수 | 예상 시간 | 예시 |
|-------------|----------|----------|------|
| **단순** | 10-15개 | 10분 | 랜딩 페이지, 간단한 CRUD |
| **중간** | 15-20개 | 15-20분 | 소규모 쇼핑몰, 커뮤니티 |
| **복잡** | 20-30개 | 20-30분 | 다중 역할 플랫폼, 결제 시스템 |
| **매우 복잡** | 30-50개 | 30-45분 | 엔터프라이즈 솔루션 |

## 2. 사용자 컨텍스트 정보 (MVP 고정값)

### 2.1 컨텍스트 정보의 중요성

사용자 컨텍스트 정보를 수집하면 **질문의 범위, 깊이, 선택지**를 맞춤화할 수 있습니다:

| 컨텍스트 정보 | 영향도 | 활용 예시 |
|-------------|--------|----------|
| **발주자 역할** | 높음 | PM → 비즈니스 중심 질문<br>개발자 → 기술 중심 질문 |
| **실무자 경력** | 높음 | 5년차 → 전문 용어 사용 가능<br>주니어 → 쉬운 용어 사용 |
| **서비스 국가** | 중간 | 한국 → 카카오/네이버<br>미국 → Google/Apple |
| **예산 규모** | 중간 | 소규모 → 필수 기능 중심<br>대규모 → 부가 기능 포함 |

### 2.2 MVP 고정값 설정

현재 MVP에서는 다음 값을 **고정**하여 개발 복잡도를 낮춥니다:

```javascript
const MVP_CONTEXT = {
  requester: {
    role: "프로젝트 담당 PM",
    technical_knowledge: "중급", // 개발 용어 부분 이해
    company_size: "소규모"
  },
  developer: {
    role: "프리랜서 개발자",
    experience_years: 5,
    expertise: ["웹 개발", "풀스택"]
  },
  service: {
    country: "한국",
    language: "ko",
    timezone: "Asia/Seoul"
  }
}
```

### 2.3 컨텍스트 기반 최적화 예시

#### 예시 1: 소셜 로그인 선택지 (국가별)

```javascript
// 한국 서비스
options: [
  "카카오 로그인 (추천)", // 한국 1위
  "네이버 로그인",
  "구글 로그인",
  "애플 로그인"
]

// 미국 서비스 (향후)
options: [
  "Google 로그인 (추천)",
  "Apple 로그인",
  "Facebook 로그인",
  "Email only"
]
```

#### 예시 2: 질문 깊이 조정 (발주자 역할별)

```javascript
// PM 발주자 (비즈니스 중심)
질문: "사용자 인증은 어떻게 하실 건가요?"
선택지:
  - 이메일로만 가입
  - 소셜 로그인 (카카오, 네이버)
  - 전화번호 인증

// 개발자 발주자 (기술 중심) (향후)
질문: "인증 방식을 선택해주세요"
선택지:
  - JWT + Email/Password
  - OAuth 2.0 (카카오, 네이버)
  - SMS OTP + JWT
```

#### 예시 3: 용어 수준 조정 (실무자 경력별)

```javascript
// 5년차 프리랜서 (전문 용어 사용 가능)
설명: "RESTful API로 구현하며, 응답 시간은 200ms 이내입니다"

// 주니어 개발자 (향후)
설명: "서버와 통신하며, 빠른 응답 속도를 제공합니다"
```

## 3. UX 플로우 설계

### 3.1 전체 프로세스

```
0단계: 컨텍스트 설정 (MVP: 고정값)
   ↓
1단계: 초기 입력 (자유 서술형)
   ↓
2단계: AI 분석 & 프로젝트 분류
   ↓
3단계: 객관식 질문 (컨텍스트 기반 동적 생성)
   ↓
4단계: 실시간 명세서 생성
   ↓

   ↓
```

```
2. 복잡도 분석
   - 단순: 기본 질문 개수의 70%
   - 중간: 기본 질문 개수의 100%
   - 복잡: 기본 질문 개수의 150%

3. 사용자 구체화 수준
   - 매우 구체적 (상세한 초기 입력): -30%
   - 보통: 0%
   - 추상적: +20%

4. 컨텍스트 조정 (MVP: 고정)
   - PM 발주자: 비즈니스 질문 +20%
   - 5년차 개발자: 기술 깊이 질문 +10%
   - 한국 서비스: 국가별 특화 질문 포함

최종 질문 개수 = 기본 * 복잡도 * 구체화 * 컨텍스트
(최소 10개, 최대 50개)
```

### 3.2 질문 스킵 로직

```

IF 사용자가 "결제 기능" 선택 = No

  - "결제 방식" 질문 스킵
```

### 5.2 컨텍스트 기반 질문 렌더링 로직

```javascript
function renderQuestion(question, context) {
  // 1. 질문 텍스트 선택
  const questionText = context.requester.role === "개발자"
    ? question.question_text.for_developer
    : question.question_text.for_pm || question.question_text.default;
  
  // 2. 옵션 설명 조정
  const options = question.options.map(opt => {
    const description = context.requester.role === "PM"
      ? opt.description.for_pm
      : opt.description.for_developer || opt.description.default;
    
    // 3. 국가별 소셜 플랫폼 필터링
    let socialPlatforms = [];
    if (opt.social_platforms_by_country) {
      socialPlatforms = opt.social_platforms_by_country[context.service.country] 
        || opt.social_platforms_by_country.default;
    }
    
    return {
      ...opt,
      description,
      socialPlatforms
    };
  });
  
  return {
    questionText,
    options
  };
}

// MVP 사용 예시
const MVP_CONTEXT = {
```
