# AI Agent Project Brief: Specifai MVP

## 1. 프로젝트 목표 (Project Goal)

이 문서는 AI LLM 에이전트(V0, Zed 등에서 사용)를 위한 프로젝트 시작 프롬프트입니다. AI-Native 요구사항 명세서 자동 생성 플랫폼 'Specifai'의 MVP(Minimum Viable Product)를 개발하는 것을 목표로 합니다.

## 2. 개발 및 서비스 환경 (Development & Service Environment)

- **프로젝트 시작**: LLM 에이전트를 통해 시작합니다.
- **버전 관리**: Github
- **데이터베이스**: Supabase
- **배포**: Vercel

## 3. 핵심 기능 및 서비스 프로세스 (Core Features & Service Process)

### 3.1. 기본 페이지 구조 (Basic Page Structure)

- 별도의 랜딩 페이지 없이, 웹사이트 접속 시 바로 메인 Home 화면으로 진입합니다.
- 페이지는 좌측 사이드바와 우측 콘텐츠 영역으로 구성됩니다 (ChatGPT, Claude와 유사한 레이아웃).

#### **좌측 사이드바 (Left Sidebar)**
- **서비스 로고**: 상단에 위치합니다.
- **플립플롭 버튼 (Flip-flop Button)**: 로고 아래에 위치하는 토글 형태의 버튼입니다.
- **New 버튼**: 새 요구사항 명세서 작성을 시작하는 버튼입니다.
- **내 산출물 목록 (My Deliverables)**: 사용자가 생성한 요구사항 명세서 목록이 표시됩니다.
- **로그인 상태 버튼**: 로그인/로그아웃 및 사용자 상태를 표시하는 버튼입니다.

#### **우측 콘텐츠 영역 (Right Content Area)**
- 초기 화면에는 **요구사항 초안 작성 폼**이 표시됩니다. 사용자가 자유롭게 프로젝트에 대한 아이디어를 입력할 수 있는 큰 텍스트 영역(textarea)과 '검토 시작' 버튼으로 구성됩니다.

### 3.2. 사용자 플로우 (User Flow)

1.  **초안 작성**: 사용자가 콘텐츠 영역의 폼에 프로젝트 요구사항 초안을 자유롭게 작성하고 '검토 시작' 버튼을 클릭합니다.
2.  **초안 검토 및 확인**: 시스템이 사용자의 입력을 분석하여 **프로젝트 유형, 사용자 역할, 프로젝트 목적, 대상 고객** 등 핵심 항목을 추출하고 사용자에게 보여주며 확인을 요청합니다.
3.  **Round 1 질의응답**: 사용자가 확인하면, 시스템은 명세서를 구체화하기 위한 **Round 1 질문 목록**을 생성하여 제시합니다.
4.  **Round 2 질의응답**: 사용자가 모든 질문에 답변하면, 시스템은 답변 내용을 분석하여 부족하거나 누락된 부분을 파악하고, 이를 보완하기 위한 **Round 2 질문 목록**을 생성하여 제시합니다.
5.  **최종 산출물 생성**: 모든 질의응답이 완료되면, 시스템은 전체 내용을 종합하고 구조화하여 **최종 요구사항 명세서**를 생성합니다.
6.  **결과 확인 및 복사**: 생성된 명세서는 콘텐츠 영역에 표시되며, 사용자는 전체 내용을 쉽게 복사할 수 있는 버튼을 통해 산출물을 가져갈 수 있습니다.

### 3.3. 확장성 요구사항 (Scalability Requirements)

시스템은 향후 서비스 확장을 고려하여 유연하고 확장 가능한 구조로 설계되어야 합니다.

1.  **업무 분야 확장 (Business Domain Scalability)**:
    *   **초기 타겟**: 소프트웨어 개발
    *   **미래 확장**: 법률, 디자인, 컨설팅 등 다양한 전문 분야로 확장될 수 있도록, 분야별 특화된 질문 셋과 프로세스를 추가할 수 있는 모듈식 아키텍처를 적용해야 합니다.

2.  **소프트웨어 개발 내 하위 분야 확장 (Sub-domain Scalability)**:
    *   **초기 타겟**: 웹 서비스, 모바일 서비스, 데스크탑 서비스
    *   **미래 확장**: 게임 개발, 임베디드 시스템, 데이터 분석 등 다른 하위 분야를 쉽게 추가할 수 있도록 설계해야 합니다. 사용자가 프로젝트 초기에 분야를 선택하면, 해당 분야에 맞는 질문과 템플릿이 제공되어야 합니다.
