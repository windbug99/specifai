# SpecClear - 8. 리스크 관리

## 8.1 주요 리스크 및 완화 전략

### 리스크 매트릭스

```
        영향도 높음 ↑
             │
    [시장 규모]  [가격 저항]
    [해자 실패]
             │
    [ChatGPT]   [위시켓 추가]
    [법적 책임]
             │
    [AI 비용]   [개발 지연]
             │
        영향도 낮음 ↓────────────→ 발생 확률 높음
```

### 상세 리스크 분석

#### 🔴 위험도 높음 (High Priority)

##### 1. 시장 규모 과대평가
```
리스크:
• 실제 수요가 예상보다 작을 수 있음
• 페인포인트가 생각보다 크지 않을 수 있음

영향도: 🔴 높음
발생 확률: 🟡 중간

완화 전략:
✅ 2주 무료 베타로 빠른 검증
✅ 10명 이상 실제 사용자 피드백
✅ 유료 전환 의향 사전 조사
✅ 최소 투자로 MVP 개발

성공 지표:
• 베타 사용자 30% 이상 유료 의향
• 월 5건 이상 유료 거래
```

##### 2. 가격 저항
```
리스크:
• 30만원이 비싸다고 느낄 수 있음
• 무료 도구(ChatGPT)와 비교

영향도: 🔴 높음
발생 확률: 🔴 높음

완화 전략:
✅ Van Westendorp 가격 조사
✅ A/B 테스트 (20/30/40만원)
✅ ROI 명확히 제시
✅ 무료 체험 제공
✅ 분할 결제 옵션

성공 지표:
• 가격 수용도 50% 이상
• CAC < LTV/3
```

##### 3. 해자 구축 실패
```
리스크:
• 데이터 수집 실패
• 네트워크 효과 미달성

영향도: 🔴 높음
발생 확률: 🟡 중간

완화 전략:
✅ MVP부터 데이터 수집 시작
✅ 인센티브 제공 (할인)
✅ 단계별 해자 구축
✅ 측정 가능한 지표 설정

성공 지표:
• 6개월 내 100건 데이터
• 12개월 내 첫 패키지 출시
```

#### 🟡 위험도 중간 (Medium Priority)

##### 4. ChatGPT 개선
```
리스크:
• OpenAI가 외주 특화 기능 추가
• 무료로 비슷한 서비스 제공

영향도: 🟡 중간
발생 확률: 🔴 높음

완화 전략:
✅ 외주 도메인 전문성 강화
✅ 한국 시장 특화
✅ 법적 효력 차별화
✅ 생태계 통합 가속

대응 시나리오:
• 데이터 해자 활용
• B2B 피벗 준비
```

##### 5. 위시켓/이랜서 기능 추가
```
리스크:
• 플랫폼이 자체 명세서 기능 개발
• 무료로 제공

영향도: 🟡 중간
발생 확률: 🟡 중간

완화 전략:
✅ 빠른 시장 선점
✅ 파트너십 먼저 제안
✅ 화이트라벨 제공 준비
✅ API 제공

대응 시나리오:
• 협력 관계 구축
• B2B SaaS 전환
```

##### 6. 법적 책임 문제
```
리스크:
• 명세서 오류로 인한 분쟁
• 법적 자문 책임

영향도: 🟡 중간
발생 확률: 🟢 낮음

완화 전략:
✅ 명확한 면책조항
✅ "도구일 뿐" 명시
✅ 보험 가입 검토
✅ 법무법인 자문

예방 조치:
• 이용약관 정비
• 책임 한계 명시
```

#### 🟢 위험도 낮음 (Low Priority)

##### 7. AI API 비용 증가
```
리스크:
• OpenAI 가격 인상
• 사용량 증가로 비용 급증

영향도: 🟢 낮음
발생 확률: 🟡 중간

완화 전략:
✅ 가격 정책에 반영
✅ 효율적 프롬프트 설계
✅ 캐싱 활용
✅ 자체 모델 검토

비용 관리:
• 사용량 모니터링
• 단계별 가격 조정
```

##### 8. 개발 지연
```
리스크:
• MVP 개발 지연
• 품질 이슈

영향도: 🟢 낮음
발생 확률: 🟡 중간

완화 전략:
✅ 초미니멀 MVP
✅ 명확한 우선순위
✅ AI 도구 활용 (Cursor)
✅ 단계적 출시

일정 관리:
• 주간 마일스톤
• 일일 체크인
```

## 8.2 중단 기준 (Kill Criteria)

### 즉시 중단 조건 🚨

#### 1. 초기 검증 실패
```
조건:
□ 2주 베타 후 10명 중 3명 미만 유료 의향
□ 평균 희망 가격 10만원 미만
□ "필요 없다"는 피드백 50% 이상

판단 시점: Week 3
결정: 즉시 중단 또는 피벗
```

#### 2. 제품-시장 적합성 부재
```
조건:
□ 3개월 내 유료 고객 10명 미만
□ NPS 0 이하
□ 재구매율 10% 미만

판단 시점: Month 3
결정: 피벗 또는 중단
```

### 재검토 조건 ⚠️

#### 3. 성장 정체
```
조건:
□ 6개월 내 월 5건 미만 거래
□ 3개월 연속 성장률 0%
□ CAC > LTV

판단 시점: Month 6
결정: 전략 수정 또는 피벗
```

#### 4. 해자 구축 실패
```
조건:
□ 6개월 후 데이터 50건 미만
□ 12개월 후 패키지 0개
□ 24개월 후 파트너십 0개

판단 시점: 각 마일스톤
결정: 해자 전략 수정
```

### 경제성 기준 💰

#### 5. 수익성 악화
```
조건:
□ CAC가 LTV의 3배 초과
□ 6개월 연속 적자
□ Burn Rate > 월 500만원

판단 시점: 매월
결정: 비용 절감 또는 투자 유치
```

#### 6. 투자 실패
```
조건:
□ 6개월 내 Seed 투자 실패
□ Runway < 3개월
□ 추가 자금 조달 불가

판단 시점: Month 6
결정: 부트스트랩 또는 중단
```

## 8.3 피벗 옵션

### Option 1: PM 도구로 피벗 (가장 유력) ⭐

#### 전환 시나리오
```
현재: 외주 명세서 → 피벗: PM 협업 도구

타겟 변경:
• Before: 외주 발주자/개발자
• After: 기업 내부 PM/개발팀

기능 조정:
• 명세서 작성 (유지)
• 프로젝트 관리 (추가)
• 팀 협업 (추가)
• Jira 연동 (추가)
```

#### 장점
- 더 큰 시장 (B2B SaaS)
- 반복 수익 모델
- 기존 기술 활용 가능

#### 단점
- 경쟁 심화
- 엔터프라이즈 세일즈 필요

### Option 2: 컨텍스트 엔지니어링 플랫폼 🤖

#### 전환 시나리오
```
현재: 명세서 작성 → 피벗: AI 컨텍스트 관리

핵심 가치:
• AI에게 제공할 도메인 지식 패키징
• PM 메모리 시스템
• 프로젝트 컨텍스트 DB

수익 모델:
• 컨텍스트 패키지 판매
• API 제공
• 엔터프라이즈 라이선스
```

#### 장점
- 트렌디한 시장
- 기술적 차별화
- 높은 가격 책정 가능

#### 단점
- 시장 교육 필요
- 개념 설명 어려움

### Option 3: 교육/컨설팅 비즈니스 📚

#### 전환 시나리오
```
현재: 제품 → 피벗: 서비스

서비스 구성:
• "명확한 명세 작성법" 온라인 강의
• 기업 워크샵
• 1:1 컨설팅
• 템플릿 판매

채널:
• ultravisionlab 활용
• 온라인 코스 플랫폼
• B2B 직접 영업
```

#### 장점
- 즉시 수익 가능
- 낮은 초기 투자
- 기존 전문성 활용

#### 단점
- 확장성 제한
- 노동 집약적
- 자동화 어려움

### Option 4: 프리랜서 매칭 플랫폼 🤝

#### 전환 시나리오
```
현재: 명세서 → 피벗: 매칭 + 명세서

통합 서비스:
• 명세서 작성
• 개발자 매칭
• 프로젝트 관리
• 결제 에스크로

수익:
• 거래 수수료 (10%)
• 프리미엄 회원
• 부가 서비스
```

#### 장점
- 네트워크 효과
- 높은 거래액
- 종합 솔루션

#### 단점
- 높은 초기 투자
- 운영 복잡도
- 기존 플레이어와 경쟁

## 8.4 위기 대응 프로토콜

### 시나리오별 대응

#### Scenario 1: 경쟁사 출현
```
상황: 대기업이 유사 서비스 출시

즉시 대응:
1. 차별화 포인트 강화
2. 고객 락인 프로그램
3. 가격 방어 전략
4. PR 강화

장기 대응:
• 니치 시장 집중
• 파트너십 가속
• M&A 검토
```

#### Scenario 2: 기술 장애
```
상황: 서비스 중단 또는 데이터 손실

즉시 대응:
1. 고객 공지
2. 임시 솔루션 제공
3. 보상 정책 실행
4. 원인 분석

예방 조치:
• 백업 시스템
• 모니터링 강화
• SLA 수립
```

#### Scenario 3: 법적 분쟁
```
상황: 고객이 명세서 오류로 소송

즉시 대응:
1. 법무팀 연결
2. 증거 보전
3. 고객 소통
4. 언론 대응

예방 조치:
• 면책 조항 강화
• 보험 가입
• 법률 검토 프로세스
```

#### Scenario 4: 자금 소진
```
상황: Runway < 3개월

즉시 대응:
1. 비용 절감 (50%)
2. 수익 집중
3. 브릿지 투자
4. M&A 타진

옵션:
• 팀 축소
• 피벗
• 자산 매각
• Soft Landing
```

## 8.5 리스크 모니터링

### 주간 체크리스트

#### 비즈니스 지표
```
□ 신규 가입자 수
□ 유료 전환율
□ 이탈률
□ NPS 스코어
□ 현금 잔액
```

#### 제품 지표
```
□ 버그 발생률
□ 서버 가동률
□ 응답 시간
□ 사용자 피드백
```

#### 시장 지표
```
□ 경쟁사 동향
□ 대체재 출현
□ 규제 변화
□ 기술 트렌드
```

### 월간 리스크 리뷰

#### 평가 항목
1. 새로운 리스크 식별
2. 기존 리스크 재평가
3. 완화 전략 효과성
4. 리소스 재배분
5. 비상 계획 업데이트

#### 의사결정 프로세스
```
리스크 감지 → 영향도 평가 → 대응 옵션 검토
→ 비용/효과 분석 → 실행 → 모니터링
```

## 8.6 성공 시나리오 계획

### 급성장 대응 (Good Problem)

#### 시나리오: 예상보다 빠른 성장
```
상황:
• 월 100건 이상 거래
• 서버 과부하
• CS 대응 지연

대응:
1. 긴급 인프라 확장
2. 임시 인력 투입
3. 자동화 가속
4. 투자 유치 가속
```

#### 기회 포착
```
• PR 극대화
• 가격 인상 검토
• 파트너십 협상력 활용
• 시장 선점 가속
```

### Exit 기회 대응

#### 인수 제안 시
```
평가 기준:
• 제안 가격 vs 자체 성장 가치
• 문화적 적합성
• 시너지 효과
• 팀 처우

최소 조건:
• 10억원 이상
• 팀 고용 승계
• 제품 독립성
```

## 8.7 결론

### 핵심 리스크 관리 원칙

1. **빠른 검증**: 2주 내 시장 검증
2. **낮은 비용**: 최소 투자로 시작
3. **명확한 기준**: 중단/피벗 기준 사전 설정
4. **유연성**: 다양한 피벗 옵션 준비
5. **모니터링**: 주간/월간 체크

### 리스크 대비 기회
```
리스크는 있지만 관리 가능한 수준
기회가 리스크보다 훨씬 큼
빠른 실행과 검증이 핵심
```
